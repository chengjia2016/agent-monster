## Data Flow Example

### User Login Sequence:
```
1. User runs CLI
2. User logs in via GitHub CLI (gh auth login)
3. CLI calls GetCurrentUser() → Receives User with ID
4. CLI calls CreateOrGetUserAccount(userID, username)
5. Judge-server creates user in database
6. CLI stores currentUser.ID
7. User navigates to Pokemon list
8. CLI calls GetUserPokemons(currentUser.ID)
9. Judge-server queries Pokemon database
10. CLI displays real data (or empty state if no Pokemon)
```
